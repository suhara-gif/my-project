# vault — セカンドブレイン運用ルール（Claude が最初に読む）

このファイルはエージェントが毎セッション必ず読み込む「常時課金の税」です。
**200 行以内・軽量**に保ち、知識そのものは書かず、置き場所だけを指し示します。

## この vault は何か

あなた（Claude）の外部記憶。プレーンな Markdown の集まりで、Obsidian では人間が、
フォルダとしては Claude が、**同じ「脳」**を見ています。DB もクラウドロックインも
なく、全ては自分が所有する平文です。

## フォルダ構成（4 つだけ）

- `raw/` … 取り込んだ素材を**無加工**で置く読み取り専用の履歴。**絶対に書き換えない**。
  記事・文字起こし・通話メモ・競合ページ、`raw/sessions/`（セッション採掘）、
  `raw/research/`（リサーチ機の出力）、`raw/maintenance/`（lint 記録）。
- `entities/` … 具体物 1 つにつき 1 ページ（クライアント / 競合 / ツール / 人物）。
- `concepts/` … 考え方 1 つにつき 1 ページ（戦略 / パターン / 学び）。
- `INDEX.md` … 玄関。compiled ページを 1 行説明付きで列挙。開かずに「何があるか」を知る。
- `syntheses/` … 週次総合の置き場（横断で見えたことを 1 枚に）。

## 書き込みルール（4 つ）

1. **1 ファイル 1 レッスン。** 冒頭に 1 行サマリを置く。
2. **重複を作らず、既存ページを更新する**（update, don't duplicate）。
3. **間違いと分かったノートは削除する。**
4. **raw/（素材）と compiled（entities/concepts）は常に分ける。**

なぜ raw/ を触らないか: 同じエージェントが同じノートを読んで書き換え続けると、
細部がぼやけ誤差が積み上がる。raw/ は地の真実。wiki はその上で賢くなる。

## リンク＝知識グラフ

2 ページを結ぶときは `[[ページ名]]` で必ずリンクする。回答は全部を走査せず、
リンクを辿って（クライアント→施策→競合…）記憶のように歩く。ページが増えるほど、
検索型は雑音が増えるが、リンク型は周囲のページが強くなる。

## 読み方ルール（お金を燃やさない）

- まず `INDEX.md` を読む。次にリンクを辿り、**必要なページだけ**開く。
  フォルダ全走査は絶対にしない。
- 大きな問いは**サブエージェントに委譲**し、別コンテキストで数十ページ読ませて、
  結論だけ 1 段落で受け取る。高価なこの部屋には決定だけを置き、書庫は置かない。
- 主張には必ず**出典（raw/ のページ）へのリンク**を添える。出典なしは信用しない。

## コンパイル（あなたの仕事）

`raw/` の新素材を読み、`entities/` と `concepts/` を更新・リンクする。
変更は必ず **diff（前後の行）**で示し、「更新した」という主張だけで済ませない。
新規ページを作ったら `INDEX.md` に 1 行足す。
