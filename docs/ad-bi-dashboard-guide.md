# ウェブ広告実績 BIダッシュボード構築手順書

Looker Studio を使って、広告実績スプレッドシートを「見やすく・状況把握しやすい」ダッシュボードにするための手順書です。貼り付け・設定すればそのまま完成する形でまとめています。

---

## 0. 結論（設計方針）

- **データソースは `日次サマリ`（縦型）シートのみを使う。**
  すでに `ad_genre`（媒体）/ `business`（事業）/ `media` / `campaign_name` と次元が列として分離され、`cost` / `cv` / `cpa` / `cv_seibishi`（整備士CV）/ `application_count`（応募）等の指標も揃った、BIに最適な「縦型（long / tidy）」になっている。
- **`daily_summary`（横型38列）はダッシュボードのソースに使わない。**
  `G_TW_費用` のように「媒体 × 商材 × 指標」を1列に詰め込んだピボット済みビューであり、BIツールでの分解・集計に向かないため。
- **変換シートの作成は不要。** `日次サマリ` をそのまま Looker Studio に直結する。
- **BIツールは Looker Studio（無料・Googleスプレッドシート直結）。** 現在のデータ規模では BigQuery 等は不要。

### `日次サマリ` の主な列（縦型）

| 区分 | 列 | 内容 |
|---|---|---|
| 次元 | `日付` | 日次 |
| 次元 | `ad_genre` | 媒体（meta / google / indeed …） |
| 次元 | `business` | 事業（人材紹介 / 出張整備） |
| 次元 | `media` | 商材・配信枠（CWA / TW / OS …） |
| 次元 | `campaign_name` `campaign_type` | キャンペーン |
| 指標 | `cost` | 費用 |
| 指標 | `cv` | コンバージョン |
| 指標 | `cv_seibishi` | 整備士CV |
| 指標 | `application_count` | 応募数 |
| 指標 | `imp` `click` | 表示・クリック |
| 参考 | `cpa` `cpc` `ctr` `cvr` | 行単位の比率（※後述の理由で集計には使わない） |

---

## 1. 重要な前提：比率は「合計を割り戻す」

`日次サマリ` は**キャンペーン単位の行**なので、CPA・CPC・CTR・CVR を **行ごとの値の平均（AVG）で集計してはいけない**。
必ず「合計どうしを割る」計算フィールドを使う。

> 例：CPA を平均すると、費用が小さいキャンペーンの極端なCPAに引っ張られて実態とずれる。
> 正しくは **総費用 ÷ 総CV**。

このため、下記の計算フィールドを Looker Studio 側で定義する。

---

## 2. データソースの接続

1. [Looker Studio](https://lookerstudio.google.com/) を開く → 「作成」→「データソース」。
2. コネクタ「**Googleスプレッドシート**」を選択。
3. 対象スプレッドシートを選び、ワークシートに **`日次サマリ`** を指定。
4. 「先頭行をヘッダーとして使用」をオン。「範囲（オプション）」は空でよい（全行取り込み）。
5. 接続後、各フィールドの**型**を確認：
   - `日付` → 日付型（`YYYY-MM-DD`）。型が文字列になっていたら、計算フィールド `TODATE` で変換するか、シート側で日付書式に統一する。
   - `cost` `cv` `cv_seibishi` `application_count` `imp` `click` → 数値型。
   - `cpa` `cpc` `ctr` `cvr`（行単位の比率列）は **集計に使わない**ので、そのままで可。

---

## 3. 計算フィールド定義（コピーして登録）

データソース編集画面で「フィールドを追加」から以下を登録する。

### 集計指標（合計を割り戻す）

```
-- CPA（集計）  ※指標の集計方法は「自動 / 計算済み」のまま
名前: CPA
式 : SUM(cost) / SUM(cv)

-- CPC（集計）
名前: CPC
式 : SUM(cost) / SUM(click)

-- CTR（集計）
名前: CTR
式 : SUM(click) / SUM(imp)

-- CVR（集計）
名前: CVR
式 : SUM(cv) / SUM(click)

-- 整備士CPA（集計）
名前: CPA_整備士
式 : SUM(cost) / SUM(cv_seibishi)

-- 応募単価（集計）
名前: CPA_応募
式 : SUM(cost) / SUM(application_count)
```

> CPA / CPC / CPA_整備士 / CPA_応募 の表示形式は「通貨（¥）」、CTR / CVR は「パーセント」に設定。

### 目標CPA と達成判定（目標CPA = ¥11,000）

```
-- 目標CPA（固定値）
名前: 目標CPA
式 : 11000

-- 目標CPA差分（マイナスなら目標達成 = 安い）
名前: CPA差分
式 : (SUM(cost) / SUM(cv)) - 11000

-- 達成ステータス（色分け用）
名前: CPAステータス
式 : CASE
       WHEN SUM(cv) = 0 THEN "CVなし"
       WHEN (SUM(cost) / SUM(cv)) <= 11000 THEN "達成"
       ELSE "未達"
     END

-- 目標比（1.0未満が達成）
名前: CPA達成率
式 : (SUM(cost) / SUM(cv)) / 11000
```

> ¥11,000 は現状「全媒体・全商材 一律」。将来、商材や媒体ごとに目標を変える場合は、別途「目標CPAマスタ」シート（`business` × `media` × `目標CPA`）を作り、Looker Studio の**データの統合（ブレンド）**で `日次サマリ` に結合して `目標CPA` を可変にする。

### 着地見込み（当月ペースからの予測）

> **Looker Studio の制約に注意。** ①`LAST_DAY` 関数は存在しない。②**1つの計算フィールド内で集計関数（`SUM` 等）と非集計の値（`CURRENT_DATE()` などの日付定数）を混在させるとエラーになる**。
> このため「`SUM(cost) ÷ 当月経過日数 × 当月総日数`」を1つの計算フィールドで書くことはできない。着地見込みは**スプレッドシート側の小さな補助ブロックで算出**するのが、正確かつ確実で推奨。

`日次サマリ` とは別の空きセル（または小さな別シート）に、当月実績とペース換算を置く。`日次サマリ` の日付列を `A`、費用列を `cost`、CV列を `cv` と仮定した例：

```
当月費用(MTD)   = SUMIFS(日次サマリ!cost列, 日次サマリ!日付列, ">="&EOMONTH(TODAY(),-1)+1, 日次サマリ!日付列, "<="&TODAY())
当月CV(MTD)      = SUMIFS(日次サマリ!cv列,   日次サマリ!日付列, ">="&EOMONTH(TODAY(),-1)+1, 日次サマリ!日付列, "<="&TODAY())
経過日数         = DAY(TODAY())
当月総日数       = DAY(EOMONTH(TODAY(),0))
費用 着地見込み  = 当月費用(MTD) / 経過日数 * 当月総日数
CV 着地見込み    = 当月CV(MTD)   / 経過日数 * 当月総日数
CPA 着地見込み   = 費用着地見込み / CV着地見込み
```

> `EOMONTH(TODAY(),0)` が当月末日、`EOMONTH(TODAY(),-1)+1` が当月初日。`DAY(EOMONTH(TODAY(),0))` で当月総日数（28/30/31）が正しく取れる。
> この補助ブロックの範囲を Looker Studio の**別データソース**として追加し、スコアカード（費用着地・CV着地・CPA着地）に使う。データが縦型の `日次サマリ` 側ではなくこの小ブロックを参照するので、集計混在エラーを回避できる。

## 4. ダッシュボード構成（1ページ）

上から下に4セクション。最上部に共通コントロールを置く。

### 共通コントロール（ページ最上部）

- **期間設定（Date range control）**：既定「今月」。
- **プルダウン**：`ad_genre`（媒体）、`business`（事業）、`media`。
- これらは全チャートに効くようにページ全体へ配置。

### ① 全体KPIと着地

- スコアカード横並び：`SUM(cost)`（費用）/ `SUM(cv)`（CV）/ `CPA` / `目標CPA`。期間設定は既定「今月」。
- スコアカード（着地見込み）：補助ブロックのデータソースから `費用 着地見込み` / `CV 着地見込み` / `CPA 着地見込み`（セクション3参照）。
- **時系列グラフ**：ディメンション `日付`、指標 `SUM(cost)`（棒）+ `CPA`（線、第2軸）。費用とCPAの推移を1枚で。
- `CPA` のスコアカードに**条件付き書式**：`CPA` > 11000 を赤、それ以下を緑。

### ② 媒体別比較

- **表**：ディメンション `ad_genre`、指標 `SUM(cost)` / `SUM(cv)` / `CPA` / `CTR` / `CVR`。
- **棒グラフ**：`ad_genre` 別 `CPA`。目標線 ¥11,000 を参照線として追加。

### ③ 商材・職種別の成果

- **表**：ディメンション `business` ＋ `media`（ドリル可）、指標 `SUM(cost)` / `SUM(cv)` / `CPA`。
- 整備士向けカード：`SUM(cv_seibishi)` / `CPA_整備士`。
- 応募を見る場合：`SUM(application_count)` / `CPA_応募`。

### ④ 目標CPA達成状況

- **表（ヒートマップ）**：ディメンション `media`（または `business`×`media`）、指標 `CPA` / `CPA差分` / `CPA達成率`。
  - `CPAステータス` または `CPA達成率` で**条件付き色分け**（達成=緑 / 未達=赤）。
- **散布図**（任意）：X=`SUM(cost)`、Y=`CPA`、参照線 Y=11000。費用規模と効率を同時に把握。

## 5. 運用メモ

- スプレッドシートに行が追加されると、Looker Studio は自動で取り込む（必要に応じてデータソースの「更新頻度」を15分等に設定）。
- 新しい媒体（`ad_genre`）や商材（`media`）が増えても、縦型ゆえにダッシュボードの修正は不要。プルダウンに自動で増える。
- `daily_summary`（横型）は人が眺める用の集計ビューとしては残してよいが、**BIのソースにはしない**。
- 目標CPAを商材・媒体別に分けたくなったら、セクション3の「目標CPAマスタ＋ブレンド」へ移行する。

## 7. データの読み解き・既知の調整

実データで判明した、`日次サマリ` の値の意味と、ダッシュボードで必要な調整。

### 次元（ディメンション）の意味

- `ad_genre` の値：
  - `google` / `meta` … 各広告プラットフォーム
  - **`aggregation`** … Indeed・求人ボックス等の「アグリゲーション型求人広告」をまとめたチャネル。**他チャネルの重複（ロールアップ）ではなく独立した実チャネル**なので、全体KPIにそのまま含めてよい。
- `business` の値：`人材紹介` / `人材派遣` / `求人サイト` / `出張整備` など。
- `media` の値：`CWA` / `TW` / `MW` / `CW` / `OS` など。空（`null`）の行も存在する（下記）。

### 整備士CPA（`cv_seibishi` / `CPA_整備士`）の正しい範囲

`整備士CV`（`cv_seibishi`）は **アグリゲーション広告（`ad_genre = aggregation`）経由の整備士応募**であり、`出張整備` 事業のことではない。
そのため `CPA_整備士 = SUM(cost) / SUM(cv_seibishi)` を**全媒体の費用で割ると桁違いに大きくなる**（実例：¥726,653）。

**対処：整備士カードに「チャート単位のフィルタ `ad_genre = aggregation`」を掛ける。**
→ 「アグリゲーション広告の費用 ÷ 整備士CV」になり、費用と分母の範囲が揃う。
（より厳密にしたい場合は、整備士向けキャンペーンを `campaign_name` で絞り込む。）

### `media = null` の行は削除しない

`media` が空の行は `人材派遣` などの**実費用**を含む。フィルタで除外すると全体費用が過少になるため、**除外しない**。
④の媒体別表で `null` が「未達（CPA高）」として出るのは実態のシグナル。気になる場合はスプレッドシート側で `media` を補完する。

### コントロールはレポートレベルに

期間設定・プルダウンは既定では**そのページのみ**に効く。全ページに効かせるには、コントロールを選択 →右クリック →「レポートレベルにする」。

### CPA棒グラフの参照線（目標 11000）

棒グラフの「スタイル」タブ →下方の「参照線」→「参照線を追加」→ 定数 11000。
見当たらない場合は**複合グラフ**に変更し、棒＝`CPA`／線＝`目標CPA`（定数11000）で水平の目標線を表示する。

### 値に小数が出る場合

`cost` や `cv` に小数（例：`1,256.46`）が出るときは、元データに**按分行（月次を日割りした等）**が混在している可能性がある。元シートの行の作られ方を確認する。
