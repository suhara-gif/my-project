# アプ子 取扱説明書（早見表）

すはらさん専用のAI秘書「アプ子」の使い方まとめ。迷ったらここを見る。

## まず、呼び出し方

Claudeアプリで「**Code**」→ リポジトリ「**my-project**」を開き、チャット欄に打つだけ。

| 打つ | 何が起きる |
|---|---|
| `/up 用件` | アプ子を呼ぶ（短縮版・おすすめ）。用件は雑でOK |
| `/my-secretary 用件` | 同じ。長い正式版 |

- 一度呼び出したら、あとは普通に話しかけるだけ（毎回`/up`は不要）。
- 新しいチャットを始めたときだけ、最初に`/up`で呼ぶ。

## コマンド早見表

| コマンド | 用途 | 使う場面 |
|---|---|---|
| **`/up 〇〇`** | 何でも相談・依頼 | todo整理、メール下書き、リサーチ、壁打ち… |
| **`/up-check`** | 広告データの日次健診 | 異常（データ欠損・CPA急騰など）がある時だけ知らせる |
| **`/up-verify 〇〇`** | 成果物を別の目で検品 | 会議資料・対外メール・数字を出す前のチェック |

## 何ができるか（例）

- **todo整理**: `/up 今日やること整理して` → カレンダー見て優先順位づけ、todoフォルダに保存
- **メール下書き**: `/up 〇〇さんへの返信書いて` → すはらさんの文体（tone.md）で下書き。送信はしない
- **リサーチ**: `/up AI広告の最新ニュース調べて` → Web検索して出典つき要約
- **広告分析**: `/up CWAの広告パフォーマンス見て` → 社内シートを分析（数字はbusiness.mdの定義で）
- **検品**: 資料を作った後 `/up-verify さっきの資料チェックして` → 🔴🟡⚪で指摘

## 中身の地図（フォルダ構成）

```
company/
├── README.md              ← この取説
├── secretary/
│   ├── secretary.md        ← アプ子の人格・口調・ルール
│   ├── todo/               ← その日のtodo
│   ├── ideas/              ← アイデア・分析メモ
│   └── nippou/             ← 日報・健診ログ
└── knowledge/              ← 事業の共通知識（数字を扱う全AIが読む）
    ├── business.md         ← サイト/媒体の略号・KPI定義・⚠️CV定義の注意
    └── tone.md             ← すはらさんの対外メール文体（実メール由来）
.claude/commands/           ← /up・/up-check・/up-verify の中身
```

## 大事な約束ごと（アプ子が守ること）

- **個人情報（パスワード・カード番号等）は覚えない・保存しない。**
- **メールの送信ボタンは押さない。** 下書きまで出して、送信はすはらさんが判断。
- **数字は捏造しない。** 読めなければ「取得できず」と正直に書く。
- **重要な成果物は検品を1回挟む**（作る係と検品係を分ける）。
- ファイルの保存・取り込み（マージ）は、日常ファイルなら**確認なしで自動**（承認済み）。

## 新しい「型」を増やすには

よく使う頼み方は、`/up 今のをコマンドにしておいて` と言えば `/my-〇〇` として登録できる。
名前を`my-`か`up`で始めると、`/up` や `/my` と打つだけで一覧に出て選べる。

## いま残っている宿題

- **毎朝の自動健診（/up-check の定期実行）**: 承認が下りず未設定。手動実行は可能。
- **広告ツールの直接接続**: 承認が通れば、アプ子がMeta/Google広告の数字を代行取得できる。

## 困ったとき

- 回答がズレてきたら → 新しいチャットを開いて`/up`から呼び直す（文脈がリセットされて精度が戻る）。
